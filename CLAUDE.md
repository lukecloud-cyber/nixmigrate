# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

NixOS flake managing two hosts: a bare-metal AMD workstation (`nixpc`) and a QEMU/KVM VM (`nixvm`). Both are x86_64-linux, pinned to nixpkgs-unstable with Home Manager as a NixOS module.

## Common Commands

```bash
# Apply config changes (pick the host matching your machine)
sudo nixos-rebuild switch --flake ~/projects/nixmigrate#nixpc
sudo nixos-rebuild switch --flake ~/projects/nixmigrate#nixvm

# Apply Sly-Harvey Hyprland rice instead
sudo nixos-rebuild switch --flake ~/projects/nixmigrate/slyhypr#nixpc
sudo nixos-rebuild switch --flake ~/projects/nixmigrate/slyhypr#nixvm

# Update flake inputs to latest
nix flake update

# Roll back if something breaks
sudo nixos-rebuild switch --rollback

# Check flake evaluation without building
nix flake check
```

## Architecture

**Entry point:** `flake.nix` defines two `nixosConfigurations` (nixpc, nixvm). Each references a host-specific `configuration.nix` and `home.nix`.

**Host configs** (`hosts/<host>/`):
- `configuration.nix` — system-level: kernel, boot, users, imports from `modules/system/`
- `home.nix` — user-level: imports from `modules/home/`
- `hardware-configuration.nix` — machine-specific (generated by `nixos-generate-config`, not hand-edited)

**Shared modules** (`modules/`):
- `system/` — NixOS modules (desktop, audio, network, hardware, gaming, virtualisation)
- `home/` — Home Manager modules (shell, cli, dev, apps, backup)

Both hosts share the same module tree. The VM (`nixvm`) omits `hardware.nix`, `gaming.nix`, `virtualisation.nix`, and `backup.nix`. Host-specific differences are controlled by which modules each host's `configuration.nix` and `home.nix` import.

**Standalone Sly-Harvey flake** (`slyhypr/`): Self-contained flake forked from [Sly-Harvey/NixOS](https://github.com/Sly-Harvey/NixOS) providing a fully-riced Hyprland desktop with Catppuccin theming, waybar, rofi, hyprlock, and custom scripts. Has its own `hosts/nixvm/` and `hosts/nixpc/` with `variables.nix` files that control desktop, bar, terminal, editor, browser, and GPU driver. KDE Plasma 6 is enabled alongside Hyprland as a fallback session.

**Dotfiles** (`dotfiles/`): Files placed into `~` by Home Manager (currently LazyVim config at `dotfiles/nvim/`).

**Scripts** (`scripts/`): `backup_home.sh` is deployed by `backup.nix` to `~/backup_home.sh`.

## Key Conventions

- Nix language only — no shell scripts for config, no imperative package management
- Home Manager is integrated as a NixOS module (not standalone), so system and user config build together
- `hardware-configuration.nix` files are machine-specific and should not be edited by hand; they are replaced with output from `nixos-generate-config` at install time
- Credentials (e.g., rclone config) are intentionally excluded and restored manually from Bitwarden
- The user account is `luke` on both hosts
